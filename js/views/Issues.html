Issues and Contribution

NASA initiated the World Wind open source project in 2003. The objectives try to achieve balance between the needs of the stakeholders and the greater community. Ongoing development of the code is primarily by NASA software engineers. Code contribution and bug fixes are greatly appreciated but require careful analysis by the NASA engineers before being incorporated into the code base.

Guidelines for reporting issues and bugs, requesting new features, and pull requests are detailed below. The World Wind team uses GitHub and TravisCI extensively for managing development of the Java, Android, and Web SDKs.

Issues and Bugs

All issue and bug tracking is now provided by GitHub's Issue tracker. You will need a GitHub account to create an issue. Providing a short, self contained code example of how the issue manifests, an image, or stack trace, significantly facilitates investigation efforts. The World Wind team will attempt to add a label to your reported issue in a timely fashion. Resolution of the issue is dependent on the team's current priorities and work load.

Feature Requests

New features should use the GitHub Issue functionality. The World Wind team will likely add a 'request' label. The World Wind team uses the GitHub Issue tracker to outline development goals, including future features. Planned future features will be labeled 'enhancement' or 'new feature'.

Pull Requests

Community participation is encouraged through the use of the GitHub issue tracker and the World Wind community forums. Pull requests are an acceptable way to propose features and fixes, and are reviewed by the NASA team as appropriate, given their perceived critical nature and available resources. Ideas and code culled from pull requests will be attributed to their authors through mentions and links in subsequent commits.
